# 🚀 LeetGPU

欢迎来到 **LeetGPU** 代码仓库！

![Static Badge](https://img.shields.io/badge/updating-green)

本项目汇集了我对 LeetCode 算法题目的个人解答与探索。建立这个仓库的初衷不仅是记录代码，更是为了深入研究不同难度题目下的**性能优化**与**解题思路**。

## 📂 仓库结构 | Repository Structure

本仓库基于[LeetGPU](https://leetgpu.com)按照题目难度分级，每个难度文件夹下均包含独立的说明文档，详细记录了该层级题目的核心算法与优化考量。

### 1. [🟢 Easy (简单)](./Easy)
> **基础与规范**
> 
> 这里包含入门级别的题目。虽然题目简单，但我重点关注代码的**可读性**与**基础优化**。
> * 👉 **详情请见：** [Easy/README.md](./Easy/README.md)（包含基础题目的具体优化策略）

### 2. [🟡 Medium (中等)](./Medium)
> **思维与技巧**
> 
> 这里是算法练习的核心区域。涵盖了大多数面试高频题，重点探讨**数据结构的选择**与**时间复杂度**的平衡。
> * 👉 **详情请见：** [Medium/README.md](./Medium/README.md)（包含中等题目的算法权衡与优化思考）

### 3. [🔴 Hard (困难)](./Hard)
> **极限与突破**
> 
> 这里包含高难度的挑战题。重点在于处理**边界情况**、**极值优化**以及**复杂算法（如DP、图论）**的深度应用。
> * 👉 **详情请见：** [Hard/README.md](./Hard/README.md)（深入剖析困难题目的深度优化与解题路径）

---

## 🌟 项目亮点 | Highlights

* **分级详解**：不同于通过单一列表展示，我为每个难度级别编写了独立的 `README`，针对性地复盘该难度下的通用解题模式。
* **优化优先**：代码不仅仅是为了“通过（Accepted）”，更关注如何让代码运行得更快、占用的内存更少。具体的优化 Benchmark 和思考过程记录在各子目录中。

## 🛠️ 使用说明 | How to Use

您可以直接进入对应的难度文件夹查看源代码。建议配合各文件夹下的 `README.md` 阅读，以获取最佳的学习体验。

```bash
# 克隆本仓库到本地
git clone [https://github.com/HershelYen/leetgpu.git](https://github.com/HershelYen/leetgpu.git)

# 进入特定难度目录（例如 Medium）
cd LeetGPU/medium
